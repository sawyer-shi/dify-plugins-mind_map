## English Version

### Overview

Mind Map Generator Plugin (hereinafter referred to as "this plugin") is committed to protecting user privacy. This privacy policy explains how we collect, use, store, and protect your data when using this plugin.

### Data Collection Statement

#### What data do we collect?

This plugin processes the following types of data during operation:

**1. User Input Content**
- **Markdown text content**: Text content provided by users for generating mind maps
- **Layout preference settings**: Layout type selected by users (center radial or horizontal expansion)
- **File name settings**: User-defined output file names (optional)

**2. Technical Runtime Data**
- **Temporary processing files**: Temporary image files generated during plugin operation
- **System font information**: System font paths detected to ensure Chinese display
- **Memory usage**: Memory consumption data during plugin operation

#### What data we do NOT collect

This plugin explicitly declares that it does **NOT collect** the following types of personal data:
- ❌ Direct identification information such as names, emails, phone numbers
- ❌ Device identification information such as IP addresses, device IDs, MAC addresses
- ❌ Location information, GPS coordinates
- ❌ Browsing history, user behavior data
- ❌ Biometric information
- ❌ Financial information
- ❌ Health data
- ❌ Any combination information that can be used to identify individuals

### Data Usage Purpose

We process user data for the sole purpose of:

1. **Function implementation**: Converting user-provided Markdown text into mind map images
2. **Technical optimization**: Ensuring the plugin operates normally in different system environments
3. **Quality assurance**: Generating high-quality PNG image output

### Data Storage and Processing

#### Local Processing
- All data processing is performed within the Dify platform's plugin runtime environment
- No user data is transmitted to any external servers
- No external network connections are established

#### Temporary Storage
- The plugin may create temporary files for image processing during operation
- All temporary files are automatically deleted after processing completion
- No persistent storage is performed

#### Memory Management
- Plugin runtime memory is limited to 100MB
- All memory resources are automatically released after processing completion
- No memory cache of user data is retained

### Third-Party Services

#### No Third-Party Data Sharing
This plugin **does not use any third-party services**, including but not limited to:
- No external API calls
- No third-party database connections
- No third-party analytics services
- No third-party authentication services integration

#### Dependency Library Declaration
This plugin uses the following open-source technical libraries, all for local processing without data transmission:
- **Matplotlib**: For graphics rendering, no network functionality
- **Pillow (PIL)**: For image processing, purely local operations
- **NumPy**: For numerical computation, no data collection functionality

### Data Security

#### Technical Protection Measures
- **Isolated execution**: Runs in Dify platform's secure sandbox environment
- **Memory limitation**: Strict 100MB memory limit to prevent resource abuse
- **Temporary file cleanup**: Automatic cleanup of all temporary processing files
- **No network access**: Plugin has no network access permissions

#### Data Lifecycle
1. **Input stage**: Receive user Markdown text
2. **Processing stage**: Parse data and generate images in memory
3. **Output stage**: Generate PNG image files
4. **Cleanup stage**: Automatically clean up all temporary data and memory

### User Rights

#### Data Control Rights
- Users have complete control over input content
- Users can choose not to use any plugin functionality
- User input data is only used for current processing and is not retained

#### Transparency Commitment
- This privacy policy is publicly transparent and distributed with the plugin
- Any changes to data processing methods will update this policy and notify users
- Users can view the plugin source code at any time to understand data processing details

### Compliance Statement

This plugin's data processing methods comply with the following standards:
- **Minimization principle**: Only processes data necessary to complete functionality
- **Purpose limitation**: Does not use data for purposes other than declared
- **Transparency**: Publicly discloses all data processing activities
- **Security**: Takes appropriate technical measures to protect data

### Privacy Policy Updates

#### Update Notifications
- If this privacy policy needs updating, users will be notified through plugin version updates
- Major changes will be clearly marked in plugin descriptions
- Users can check plugin versions to understand the latest privacy policy status

#### Version History
